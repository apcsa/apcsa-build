Here is one way to describe an alarm clock:

"An alarm clock has many different 'sound the alarm' behaviors, one for each possible time that it can sound the alarm. Thus there is a 'sound the alarm at 5am' behavior, a 'sound the alarm at 5:01' behavior, and so forth."

This description uses no information, only behaviors. The behaviors are quite similar; they're all of the form:

"Sound the alarm at ______ ."

Here is a better description that turns the "______" into information:

"An alarm clock has one 'sound the alarm' behavior, and one variable that remembers what time the alarm should go off."

The use of the information "what time the alarm should go off" has simplified the clock's description a lot. This holds in general: behaviors that use information in the object's state simplifies an object's description.
