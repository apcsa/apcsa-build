Each variable has a name and a value.

Behaviors are actions that the object can perform; state consists of information (variables) that the object can remember.

A programming object can keep track of many different pieces of information by containing many different variables. We refer to each variable by its name. Each variable will also have a value, the thing that is being remembered. (The value may vary—it might be one thing at one point in a program, and another thing at another point—which is why we use the term "variable".)

This information—the values of all the variables taken together—is called the object's state.



It is useful to imagine that the variable is a box. The “thing” inside the box is its value at any particular point in time.
